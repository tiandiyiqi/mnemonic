# 产品简报：Mnemonic - 通用 Agent 记忆系统

## 问题陈述

### 表面问题
AI Agent 在会话结束后无法记住用户偏好和项目决策，导致用户需要重复说明相同的需求。

### 真实问题（5 Whys 分析）
通过 5 Whys 根因分析，我们发现真实问题是：

1. **上下文窗口限制** - AI Agent 的上下文窗口有限，无法持久化所有历史信息
2. **跨会话知识丢失** - 会话结束后，所有偏好、决策和经验都会丢失
3. **跨项目知识隔离** - 用户的全局偏好（如编码风格）无法在多个项目间共享
4. **缺乏学习能力** - AI 无法从历史成功和失败中学习，会重复相同的错误
5. **静态配置不足** - 现有的配置文件（如 .editorconfig）是静态的，无法记录动态的决策过程和经验教训

**核心痛点：** AI Agent 缺乏"记忆"，无法像人类一样积累经验、学习偏好、避免重复错误。

- 使用频率：每天多次使用 AI Agent（Trae、Claude Code、Cursor 等）

**目标：**

- 减少重复性的偏好说明
- 让 AI 记住项目的技术决策
- 从历史经验中学习，避免重复错误

**痛点：**
- 每次新会话都要重新说明编码风格偏好（如"我喜欢函数式编程"）
- AI 在不同项目中给出不一致的建议
- AI 会重复之前已经失败的方案（如"上次用过时的 API 失败了"）
- 无法跨项目复用成功的技术方案

**典型场景（JTBD）：**
- 当我在**开始新的编码会话**时，我想要 AI 自动应用我的编码偏好，以便**我不需要重复说明**

- 当我在**重构代码**时，我想要 AI 提醒之前的失败教训，以便**避免重复错误**
- 当我在**新项目中遇到类似问题**时，我想要 AI 推荐其他项目的成功方案，以便**快速解决问题**

**痛点：**

- 项目的技术决策（如"使用 PostgreSQL"）没有被 AI 记住
- 新成员不了解项目的历史决策和约束

**典型场景（JTBD）：**

## 解决方案概述

Mnemonic 是一个为 AI Agent 设计的**通用记忆系统**，提供 **全局 + 项目** 双层存储架构，让 AI Agent 拥有持久记忆能力。

### 核心能力

1. **双层存储架构**
   - **全局记忆（Global）** - 跨项目共享的用户偏好和约束（如"我喜欢函数式编程"）
   - **项目记忆（Project）** - 项目特定的决策和上下文（如"这个项目使用 PostgreSQL"）

2. **WAL 写入协议**
   - 使用 Write-Ahead Logging 确保记忆的完整性和可追溯性
   - 结构化格式，支持审计和回溯

3. **语义检索引擎**
   - 跨层级搜索相关记忆
   - 智能匹配用户偏好、约束和历史决策

4. **上下文感知推荐（Perceive）**
   - 基于当前对话自动推荐相关记忆
   - 主动预防重复错误，复用成功经验

5. **自我反思机制（Reflect）**
   - 任务完成后记录洞察与经验
   - 实现持续学习与能力进化

6. **零依赖设计**
   - 纯 Python 实现，无需外部数据库
   - 轻量级，易于集成到任何 AI Agent

### 技术架构

```
~/.mnemonic/                    # 全局记忆 (跨项目共享)
├── RULES.md                    # 全局偏好与约束
├── JOURNAL/                    # 全局反思日志 (YYYY-MM-DD.md)
└── audit.jsonl                 # 全局审计日志

项目目录/.mnemonic/              # 项目记忆 (项目特定)
├── SESSION.md                  # 项目会话上下文
├── DECISIONS.md                # 项目决策记录
└── audit.jsonl                 # 项目审计日志
```

### 记忆路由规则

| 记忆类型 | 存储层 | 说明 | 示例 |
|---------|-------|------|------|
| `preference` | 全局 COLD | 用户偏好，跨项目生效 | "我喜欢函数式编程" |
| `constraint` | 全局 COLD | 约束条件，跨项目生效 | "不要使用 var 关键字" |
| `decision` | 项目 HOT | 项目决策，仅当前项目 | "使用 PostgreSQL 作为数据库" |
| `correction` | 项目 HOT | 纠正记录，项目特定 | "上次用错了 API 版本" |
| `reflect` | 全局 WARM | 反思日志，跨项目学习 | "重构登录模块失败：使用了过时的 API" |

## 核心功能

### MUST（必须实现）

1. **remember（存储记忆）**
   - 自动识别记忆类型（preference/constraint/decision/correction）
   - 智能路由到全局或项目存储
   - WAL 格式写入，确保数据完整性

2. **search（搜索记忆）**
   - 跨层级搜索（项目优先，然后全局）
   - 关键词匹配和语义检索
   - 返回相关性排序的结果

3. **perceive（主动感知）**
   - 基于当前对话上下文提取关键词
   - 自动搜索相关记忆
   - 生成可操作的推荐（如"上次重构登录时使用了过时 API，请检查 API 版本"）

4. **reflect（自我反思）**
   - 任务完成后记录洞察
   - 记录成功经验和失败教训
   - 支持置信度评分

5. **CLI 工具（mnemonic-cli）**
   - 一键安装到 40+ 种 AI Agent（Trae、Claude Code、Cursor、Windsurf 等）
   - 支持全局和项目级安装
   - 自动检测已安装的 Agent

### SHOULD（应该实现）

6. **migrate（数据迁移）**
   - 从旧版数据格式迁移到新架构
   - 保留历史数据

7. **时间衰减算法**
   - 近期记忆权重更高
   - 旧记忆逐渐降低相关性

8. **多语言支持**
   - 中英文关键词提取
   - 中英文停用词过滤

### COULD（可以实现）

9. **记忆归档**
   - 自动归档过期记忆
   - 减少搜索空间，提高性能

10. **记忆可视化**
    - 展示记忆的时间线
    - 可视化记忆的关联关系

11. **团队共享**
    - 导出/导入项目记忆
    - 团队成员共享项目决策

## 成功指标（SMART）

### 1. 用户效率提升
- **指标：** 用户重复说明偏好的次数减少 80%
- **衡量方式：** 统计用户在会话中重复说明相同偏好的次数
- **时间：** 3 个月内达成

### 2. 错误预防
- **指标：** AI 推荐历史失败方案的次数减少 70%
- **衡量方式：** 统计 perceive 功能成功推荐相关失败教训的次数
- **时间：** 6 个月内达成

### 3. 用户采用率
- **指标：** 100+ 用户安装并使用 Mnemonic
- **衡量方式：** 统计 CLI 安装次数和活跃用户数
- **时间：** 6 个月内达成

### 4. 记忆准确性
- **指标：** perceive 推荐的相关性评分 > 0.8
- **衡量方式：** 用户反馈和相关性算法评分
- **时间：** 持续优化

### 5. 跨项目知识复用
- **指标：** 50% 的用户在多个项目中使用全局记忆
- **衡量方式：** 统计全局记忆的读取次数和项目数量
- **时间：** 6 个月内达成

## 风险与依赖

### 风险

1. **隐私风险**
   - 记忆存储在本地文件系统，可能包含敏感信息
   - **缓解措施：** 明确告知用户数据存储位置，建议不要记录敏感信息

2. **记忆冲突**
   - 全局偏好和项目决策可能冲突（如全局偏好 TypeScript，但项目使用 JavaScript）
   - **缓解措施：** 项目记忆优先级高于全局记忆

3. **记忆膨胀**
   - 长期使用后，记忆文件可能变得很大，影响搜索性能
   - **缓解措施：** 实现时间衰减和归档机制

4. **AI Agent 兼容性**
   - 不同 AI Agent 的集成方式不同，可能需要适配
   - **缓解措施：** 提供通用的 CLI 接口，支持 40+ 种 Agent

### 依赖

1. **Python 3.8+**
   - 核心记忆系统使用 Python 实现
   - 无外部依赖，降低安装门槛

2. **Node.js 18+**
   - CLI 工具使用 TypeScript/Node.js 实现
   - 依赖 yargs、chalk、prompts 等库

3. **AI Agent 支持**
   - 需要 AI Agent 支持自定义 Skill/Tool
   - 目前支持 Trae、Claude Code、Cursor、Windsurf 等

## 约束条件

1. **零外部依赖**
   - 核心记忆系统不依赖外部数据库或服务
   - 所有数据存储在本地文件系统

2. **轻量级设计**
   - 记忆文件使用 Markdown 格式，人类可读
   - 审计日志使用 JSONL 格式，易于解析

3. **向后兼容**
   - 提供数据迁移工具，支持从旧版本升级
   - 保留历史数据

4. **隐私优先**
   - 所有数据存储在本地，不上传到云端
   - 用户完全控制自己的记忆数据

5. **跨平台支持**
   - 支持 macOS、Linux、Windows
   - 使用跨平台的文件路径处理
